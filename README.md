# LLM From Scratch - 한글 단어 생성 모델

한국어 단어 데이터를 사용한 언어 모델 구현 (바이그램 → 트라이그램 → N-gram 과적합 분석)

## 📁 프로젝트 구조

### Step 01: 글자 빈도 분석
- **step01_charfreq.c** - 한글 글자 빈도수 분석 프로그램
- **step01_output.txt** - 빈도수 분석 결과 (상위 100개 글자)

### Step 02: 바이그램 단어 생성기
- **step02_wordgen.c** - 바이그램 모델 기반 단어 생성기
- **step02_bigram_stats.txt** - 바이그램 학습 통계 (159,552개 고유 바이그램)
- **step02_frequency.txt** - 생성된 단어 빈도 분석
- **step02_output.txt** - 100개 생성된 단어 목록

**특징:**
- EOS (End Of Sequence) 토큰 사용
- 확률 기반 다음 글자 샘플링
- 단어 시작 글자 빈도 학습

### Step 03: 트라이그램 단어 생성기 + 단어 검증
- **step03_trigram.c** - 트라이그램 모델 기반 단어 생성기
- **step03_output.txt** - 100개 생성된 단어 + 검증 결과

**특징:**
- BOS/EOS (Begin/End Of Sequence) 토큰 사용
- 트라이그램 (c1, c2) → c3 패턴 학습
- **단어 검증 기능**: [O] 실제 단어, [X] 생성된 단어
- 해시 테이블 기반 효율적 구현
- 3만 개 단어 학습 → 실제 단어 37%, 생성 단어 63%

### Step 04: 바이그램 vs 트라이그램 비교
- **step04_compare_models.c** - 두 모델 비교 프로그램
- **step04_compare_results.txt** - 비교 결과 및 통계

**비교 결과:**
```
           바이그램      트라이그램
실제단어    33 (33.0%)    42 (42.0%)
생성단어    67 (67.0%)    58 (58.0%)
```

**결론:**
- **바이그램**: 더 창의적 (새로운 단어 생성 67%)
- **트라이그램**: 더 정확함 (실제 단어 비율 42%)

### Step 05: N-gram 과적합(Overfitting) 분석 ⭐
- **step05_overfitting_analysis.c** - 2~6-gram 종합 분석 프로그램
- **step05_overfitting_results.txt** - 과적합 분석 결과
- **analyze_word_length.c** - 단어 길이 분포 분석

**핵심 발견:**
```
N-gram  사전일치[O]  생성단어[X]  평균길이
2       39.0%       61.0%       3.07글자
3       35.0%       65.0%       3.27글자
4       52.0%       48.0%       3.08글자  ⚠️ 과적합 peak
5       44.0%       56.0%       3.13글자
6       37.0%       63.0%       3.46글자
```

**중요한 통찰:**
- **평균 단어 길이: 3.21 글자**
- **과적합 전환점: 4-gram**
  - 3글자 문맥 = 거의 단어 전체를 문맥으로 사용
  - 결과: 학습 데이터를 거의 그대로 재생산 (52% 일치)
- **5-gram 이상: 데이터 희소성 문제**
  - 문맥이 너무 길어서 매칭 패턴 부족
  - 성능 오히려 하락

**100-gram이 불가능한 이유:**
- 평균 단어가 3.21글자인데 99글자 문맥은 물리적으로 불가능
- N > 평균 단어 길이면 의미 없음

## 📊 주요 발견

### 1. N-gram과 과적합의 관계

**가설:**
- N ↑ → 사전 일치율 ↑ → 100% 수렴 (과적합)

**실제 결과:**
- N이 평균 단어 길이에 가까울 때 과적합 최대 (4-gram = 52%)
- N이 너무 크면 오히려 데이터 희소성으로 성능 하락

**과적합 메커니즘:**
```
2-gram: "가" → ? (자유로움)
3-gram: "가족" → ? (적당함)
4-gram: [BOS][가][족] → ? (단어 거의 결정됨)
5-gram: [BOS][BOS][가][족] → ? (패턴 부족)
```

### 2. 최적 N-gram 선택 가이드

| N-gram | 사전일치 | 용도 |
|--------|---------|------|
| 2-gram | 39% | 창의적 단어 생성 (게임, 브랜드명) |
| 3-gram | 35% | 균형잡힌 생성 (일반적 용도) ⭐ |
| 4-gram | 52% | 안정적이지만 과적합 위험 ⚠️ |
| 5-gram+ | 불안정 | 실용성 낮음 |

### 3. 트라이그램 생성 예시
```
  1. 가통하다    [X]  (생성)
  2. 간묵        [O]  (실제)
  3. 개서기      [X]  (생성)
  9. 가속도      [X]  (생성)
 11. 경확        [O]  (실제)
```

## 🎓 배운 점

1. **N-gram은 만능이 아니다**
   - N이 크다고 항상 좋은 것이 아님
   - 데이터 특성(평균 길이)을 고려해야 함

2. **과적합의 실체**
   - 단순 암기와 패턴 학습의 경계
   - 4-gram에서 극명하게 나타남

3. **창의성 vs 정확성 트레이드오프**
   - 완벽한 모델은 없음
   - 용도에 맞는 N 선택이 중요
