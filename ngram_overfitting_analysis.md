# N-gram 과적합 분석 보고서

## 실험 개요
- **목적**: 토큰 단위 N-gram 모델에서 N이 증가할 때 과적합(overfitting) 현상 분석
- **학습 데이터**: korean_novel_sample.txt (13편 한국 고전 소설, ~200문장)
- **토큰화 방법**: Kiwi 형태소 분석기 (형태소/품사 쌍 + `<S>` 공백 토큰)
- **실험 범위**: 2-gram ~ 10-gram
- **생성 문장 수**: 각 N당 20개 (총 180개)

## 정성적 분석

### 2-3 gram: 창의적이지만 문법 취약
- **특징**: 다양한 조합, 예측 불가능
- **문제**: 문법 오류, 의미 불명확
- **예시**: "밤이 다시 있었다", "어느 날 점순이를 타었다"
- **평가**: 언어 모델로서 실용성 낮음

### 4-5 gram: 최적 균형점
- **특징**: 문법적으로 올바르고 자연스러움
- **장점**: 새로운 문장 생성 + 문법 정확도 높음
- **예시**: "나는 점순이를 좋아하었다", "소녀는 결국 세상을 떠나었다"
- **평가**: 실용적 언어 모델의 최적점
- **한계**: 일부 원문 문장이 나타나기 시작

### 6-7 gram: 과적합 시작
- **특징**: 대부분 원문 문장
- **문제**: 창의성 급감, 원문 암기 시작
- **예시**:
  - 6-gram: "그러나 한 오십 세는 되었을까" (운수 좋은 날 원문)
  - 7-gram: "그런데 오늘은 좋은 날이었다" (운수 좋은 날 원문)
- **평가**: 생성 모델보다는 검색 모델에 가까워짐

### 8-10 gram: 완전한 과적합
- **특징**: 거의 모든 문장이 원문 그대로
- **문제**: 새로운 문장 생성 불가능
- **예시**:
  - 9-gram: "이날이야말로 동소문 안에서 인력거꾼 노릇을 하는 김 첨지에게는 오래간만에도 닥친 운수 좋은 날이었다" (완전 일치)
  - 10-gram: "조선총독부 앞에서 승객을 태우어다 흥화문까지 가는데 승객은 남자이었다" (완전 일치)
- **평가**: 학습 데이터 암기, 일반화 능력 상실

## 핵심 인사이트

### 1. 최적 N 선택의 중요성
- **너무 작은 N (2-3)**: 문법 정확도 낮음
- **적절한 N (4-5)**: 문법 정확 + 일반화 능력
- **너무 큰 N (6+)**: 과적합으로 암기만 수행

### 2. 데이터 희소성(Data Sparsity) 문제
- 학습 데이터: ~200문장 → N이 커질수록 동일 패턴 재등장
- 해결책: 더 많은 학습 데이터 필요 (수만~수십만 문장)

### 3. 조합 폭발(Combinatorial Explosion)
- 토큰 수: 591개
- 가능한 N-gram 조합:
  - 2-gram: 591² = 349,281
  - 5-gram: 591⁵ = 7.14 × 10¹³
  - 10-gram: 591¹⁰ = 5.11 × 10²⁷
- 실제 학습된 N-gram: ~5,000개 (0.000001%)

### 4. 트레이드오프: 문맥 vs 일반화
- **긴 문맥 (큰 N)**: 정확한 문맥 이해, 하지만 과적합
- **짧은 문맥 (작은 N)**: 일반화 능력, 하지만 문법 오류
- **균형점**: 한국어 토큰 단위에서 **4-5 gram**이 최적

### 5. 현대 LLM과의 비교
- **N-gram 모델**: 고정된 짧은 문맥 (2-10 토큰)
- **Transformer 모델**: 가변적 긴 문맥 (수천 토큰)
- **차이점**:
  - N-gram은 정확한 일치만 기억
  - Transformer는 의미적 유사성 학습
  - N-gram은 조합 폭발 문제
  - Transformer는 임베딩으로 일반화

## 결론

이 실험은 N-gram 모델의 근본적 한계를 명확히 보여줍니다:

1. **작은 데이터셋 + 큰 N = 과적합**
2. **최적 N은 데이터 규모에 의존적**
3. **4-5 gram이 한국어 토큰 기반 생성의 실용적 상한**
4. **현대 LLM(GPT, BERT 등)이 필요한 이유**: N-gram의 조합 폭발과 과적합 문제 해결

## 다음 단계 제안

1. **더 큰 데이터셋**: 수만 문장으로 확장 → 6-7 gram 일반화 능력 테스트
2. **스무딩(Smoothing)**: 미학습 N-gram 처리 (Kneser-Ney, Laplace)
3. **백오프(Backoff)**: N-gram → (N-1)-gram 대체 전략
4. **신경망 N-gram**: 임베딩 레이어로 의미적 유사성 학습
5. **Transformer 구현**: Self-attention으로 진정한 문맥 이해
